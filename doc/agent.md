# agent 部分开发计划

- 独立于 recall 模块，单开一个 agent 模块，独立进程，独立前端+后端

- 所有记忆使用markdown，直接作为上下文，先不使用向量数据库

## 架构

- 分为两部分
    - 固定时间间隔的 heart beat
    - 主动响应用户的消息

- 分层
    - 随机想法
        - 模拟随机的情绪波动
        - 想法分为两部分，一部分预设好的想法，一部分随机产生（LLM调用）
        - 从想法中随机选出一个，传递给下一层
        - 上下文：无
        - 每次意图完成都重新随机
    - 内心活动
        - 上一层的情绪是主基调
        - 这部分调用 llm + 相关信息的搜索，形成意图
        - 实现意图后返回这一层，随机产生新的意图
    - 实现意图
        - 拿到意图
        - 调用工具实现意图（主要是用户数据查询） + 与用户对话

- 记忆
    - 长期记忆
        - 全局可见，每一层都能看到长期记忆
    - 短期记忆
        - 单个 heart beat 内上下文

## 架构

- 分层
    - 数据库整理
        - 数据库中只有 ocr+文本，这一层应该给数据库写入 summary，分析用户在做什么
        - 查 summary，找缺少 summary 的时间
        - 分段拉取 ocr 与其他信息，数据库写入总结
        - 如果有不清晰的信息，直接看图片
    - 情绪层
        - heart beat 开始时触发一次
        - 与上一次 heart beat 距离时间 > dt，随机选择
        - 否则复用上一次

    - 交互层
