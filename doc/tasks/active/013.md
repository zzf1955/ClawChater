## 目标

消除三个 Monitor 之间的重复代码，统一设置读取和解析风格；合并重复的时间解析逻辑。

## 范围

| # | 问题 | 文件 |
|---|------|------|
| 1 | `_read_setting()` 重复 | `screen_monitor.py:44`, `time_monitor.py:40` |
| 2 | `_parse_positive_float()` 重复 | `screen_monitor.py:54`, `time_monitor.py:50` |
| 3 | `_parse_positive_int()` 仅 screen 有 | `screen_monitor.py:64` |
| 4 | `resource_monitor` 设置读取风格不一致 | `resource_monitor.py:138` |
| 5 | `_parse_iso8601` / `_parse_time` 重复 | `routes.py:19`, `summary.py:10` |

## 实施计划（分步骤）

1. 新建 `recall/services/monitor/utils.py`
   - 提取公共函数：`read_setting(key, db_path, setting_reader)`、`parse_positive_float(raw, default)`、`parse_positive_int(raw, default)`

2. 重构 `recall/services/monitor/screen_monitor.py`
   - 删除本地 `_read_setting`、`_parse_positive_float`、`_parse_positive_int`
   - 改为从 `utils` import

3. 重构 `recall/services/monitor/time_monitor.py`
   - 删除本地 `_read_setting`、`_parse_positive_float`
   - 改为从 `utils` import

4. 重构 `recall/services/monitor/resource_monitor.py`（可选）
   - `_read_threshold` 改用公共 `read_setting()` + 已有的 `_parse_threshold()`
   - 视改动量决定是否纳入

5. 新建 `recall/utils/time_parse.py`
   - 提取 ISO8601 解析函数 `parse_iso8601(value: str) -> datetime`
   - `recall/api/routes.py` 的 `_parse_iso8601` 改为 import 并包装 HTTPException
   - `recall/db/summary.py` 的 `_parse_time` 改为 import 并包装 ValueError

## 验收标准

- 三个 monitor 中无重复的 `_read_setting` / `_parse_positive_*` 方法
- 时间解析逻辑只有一份核心实现
- 现有测试全部通过，无回归

## 测试要求

- 运行全量测试：`python -m pytest recall/tests/ -v`
- 重点关注 monitor 相关测试、API route 测试、summary 测试
- 可选：为 `utils.py` 和 `time_parse.py` 新增单元测试
