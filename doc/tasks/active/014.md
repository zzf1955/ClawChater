## 背景

项目在 macOS 和 Windows 上交替开发，但截图和 CPU 监控两个核心功能目前仅在 macOS 上可用：
- 截图依赖 `screencapture` 命令，Windows 上返回假数据
- CPU 监控依赖 `os.getloadavg()`，Windows 上降级返回 0.0

## 目标

在 Windows 上实现真实的截图捕获和 CPU 使用率采样，使核心功能跨平台可用。

## 范围

| # | 内容 | 文件 |
|---|------|------|
| 1 | Windows 截图实现 | `recall/services/capture.py` |
| 2 | Windows CPU 使用率采样 | `recall/services/monitor/resource_monitor.py` |

不包含：GPU 监控（`nvidia-smi` 本身跨平台）、OCR（`tesseract` 本身跨平台）。

## 实施计划（分步骤）

### 1. Windows 截图 (`capture.py`)

修改 `_default_screenshot_provider`，在 `platform.system() == "Windows"` 分支中使用 `PIL.ImageGrab`：

```python
from PIL import ImageGrab
img = ImageGrab.grab()
buf = BytesIO()
img.save(buf, format="JPEG")
payload = buf.getvalue()
```

`Pillow` 已在 `requirements.txt` 中，无需新增依赖。

### 2. Windows CPU 监控 (`resource_monitor.py`)

修改 `_sample_cpu_usage`，在 `os.getloadavg()` 抛出 `OSError` 时使用 `psutil`（可选依赖）或 WMI 查询作为后备：

方案 A（推荐）— 使用 `psutil`：
```python
import psutil
return psutil.cpu_percent(interval=0.5)
```

方案 B — 无额外依赖，使用 `wmic`：
```python
output = subprocess.check_output(
    ["wmic", "cpu", "get", "loadpercentage"],
    text=True, timeout=2.0,
)
```

### 3. 测试

- 为截图和 CPU 采样各自编写 mock 测试，验证 Windows 分支逻辑
- 确保 macOS 行为不受影响

## 验收标准

- Windows 上 `CaptureService.capture()` 生成真实 JPEG 截图文件
- Windows 上 `_sample_cpu_usage()` 返回有效的 CPU 使用率百分比（非固定 0.0）
- macOS 上原有行为无回归
- 现有测试全部通过
