## 背景

前端和其他 agent 需要通过统一 REST API 访问截图、OCR、摘要与配置数据，且应用启动需同时管理 DB 与 Engine 生命周期。

## 目标

完成 `api/schemas.py`、`api/routes.py` 与 `app.py` 生命周期编排，提供文档约定的全部接口。

## 范围

- 包含：截图查询接口、图片文件接口、摘要查询与写入接口、配置查询与更新接口、启动/关闭钩子。
- 不包含：鉴权体系、外部多租户、复杂限流与审计系统。

## 实施计划（分步骤）

1. 设计 Pydantic schema 与参数校验规则，统一请求/响应结构。
2. 实现 `routes.py`，直接调用 `db/*` 模块完成 CRUD。
3. 在 `app.py` 中挂载路由并接入 DB 初始化、Engine 启停钩子。
4. 增加错误处理与返回码约定，保证接口行为稳定可预期。

## 验收标准

- 重构文档列出的 API 均可访问并返回正确结构。
- `GET /api/screenshots/{id}/image` 可返回图片二进制内容。
- 启动时可完成 DB 初始化并拉起 Engine，关闭时可安全释放任务。
- 接口异常场景有明确错误码与错误信息。

## 测试要求

- 使用 FastAPI TestClient 编写接口集成测试，覆盖所有路由。
- 覆盖参数校验失败、记录不存在、文件缺失等错误路径。
- 增加生命周期测试，验证启动/关闭钩子执行顺序与幂等性。
