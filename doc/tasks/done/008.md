## 背景

重构链路跨越 monitor、capture、OCR、API、frontend，多模块并发运行，若缺少系统级验证容易在集成阶段出现回归。

## 目标

建立覆盖核心链路的自动化测试与联调基线，确保主流程“触发 -> 截图 -> OCR -> 查询”稳定可回归。

## 范围

- 包含：后端集成测试、前端 smoke 测试、端到端场景脚本、基础 CI 执行命令。
- 不包含：大规模压测平台、云端部署流水线、复杂监控告警体系。

## 实施计划（分步骤）

1. 梳理关键用户路径并定义最小回归用例集。
2. 编写后端集成测试，覆盖事件触发、数据库状态流转与 API 查询。
3. 编写前端 smoke 与端到端脚本，验证主页面关键操作。
4. 整理统一测试入口与执行文档，形成可复用回归流程。

## 验收标准

- 核心链路具备可重复执行的自动化测试。
- 回归用例可在本地一键执行并得到明确通过/失败结果。
- 关键失败日志可定位到具体模块或阶段。
- 新增功能可按既有基线快速补充测试。

## 测试要求

- 后端：`pytest` 覆盖核心服务与 API 集成场景。
- 前端：`vitest` + E2E smoke（如 Playwright/Cypress 任选其一）。
- 文档：补充测试运行说明与最小环境要求。
