## 目标

实现数据库与文件系统的双向对齐功能，解决 DB 中存在孤儿记录（文件已删除）和磁盘有文件但 DB 无记录的问题。

## 范围

### 后端
- `recall/db/screenshot.py` — 新增批量查询/删除/插入函数
- `recall/services/sync.py` — 新建，核心对齐逻辑
- `recall/api/routes.py` — 新增 `POST /api/sync` 端点

### 前端
- `recall/frontend/src/api.ts` — 新增 `syncDatabase()` API 函数
- `recall/frontend/src/App.tsx` — ConfigPage 中添加 Sync Database 按钮与结果展示

## 实施计划（分步骤）

### Step 1: DB 层新增函数
- `list_all_file_paths()` — 查询 screenshots 表所有 (id, file_path)
- `delete_screenshots_by_ids(ids)` — 批量删除记录
- `file_path_exists(file_path)` — 检查 file_path 是否已存在
- `bulk_insert_screenshots(records)` — 批量插入新记录

### Step 2: 新建 sync 服务
- `recall/services/sync.py`
- `sync_db_with_filesystem()` 函数：
  1. 查询 DB 所有记录，检查文件是否存在，收集需删除的 id
  2. 扫描 SCREENSHOTS_DIR 下所有 .jpg，与 DB file_path 集合比对，收集需导入的文件
  3. 从文件名解析 captured_at（格式 YYYYMMDDHHMMSSffffff.jpg）
  4. 执行批量删除和批量插入
  5. 返回统计结果 {deleted, imported, total_db, total_files}

### Step 3: 新增 API 端点
- `POST /api/sync` 调用 sync 服务，返回 JSON 统计结果

### Step 4: 前端集成
- `api.ts` 新增 `syncDatabase()` 函数
- ConfigPage 添加 "Sync Database" 按钮
- 点击后显示 loading，完成后展示 deleted/imported 统计

## 验收标准

1. `POST /api/sync` 正确返回 `{deleted, imported, total_db, total_files}`
2. DB 中不存在对应文件已缺失的记录
3. SCREENSHOTS_DIR 下所有 .jpg 文件均有对应 DB 记录
4. 新导入的记录 `ocr_status=pending`，后续会被 OCR worker 自动处理
5. 前端 /config 页面可点击按钮执行对齐，显示结果

## 测试要求

1. 手动验证：删除几张图片后调用 sync，确认 DB 记录被清理
2. 手动验证：往 screenshots 目录放入新图片后调用 sync，确认被导入 DB
3. 前端点击 Sync 按钮，确认 UI 正确显示结果
