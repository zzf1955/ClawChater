## 背景

新架构以 SQLite 作为核心共享状态存储，截图、OCR、摘要与动态配置都依赖稳定的数据层实现。

## 目标

完成 `schema.sql`、连接管理与 `screenshot/summary/setting` 三类 CRUD，支持首次初始化与默认配置写入。

## 范围

- 包含：建表 SQL、索引、数据库连接封装、事务与错误处理、默认 `settings` 初始化。
- 不包含：HTTP 路由、Engine 调度逻辑、前端数据展示。

## 实施计划（分步骤）

1. 设计并实现 `db/schema.sql`，覆盖三张核心表与索引。
2. 实现 `db/database.py`，提供连接、建库、初始化与上下文管理能力。
3. 实现 `db/screenshot.py`、`db/summary.py`、`db/setting.py` 的查询与写入接口。
4. 增加默认配置种子写入逻辑，确保重复启动幂等。

## 验收标准

- 启动后可自动创建 `recall.db` 与全部表结构。
- 三类 CRUD 接口可被上层服务直接调用。
- `settings` 默认值（阈值、批大小、间隔）可正确读取与更新。
- `pending/done/error` 状态流转所需字段与索引可用。

## 测试要求

- 为三类 CRUD 分别编写单元测试，覆盖成功与异常路径。
- 使用临时 SQLite 文件进行集成测试，验证初始化幂等。
- 增加索引/查询相关的基础性能 smoke 测试（小样本即可）。
