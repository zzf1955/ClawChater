## 背景

重构方案已固定前端技术栈为 React + Vite + Tailwind，需要先交付最小可用界面支撑查看与配置闭环。

## 目标

完成前端 MVP：可查询截图与 OCR 结果、查看图片、编辑动态配置、查询摘要列表。

## 范围

- 包含：页面骨架、API 请求层、筛选与列表展示、配置表单、基础错误与加载状态。
- 不包含：复杂视觉主题、权限系统、离线缓存与高级交互动画。

## 实施计划（分步骤）

1. 初始化页面结构与路由，划分截图/配置/摘要三个功能区。
2. 实现 API client 与状态管理，统一处理请求错误与加载反馈。
3. 实现截图列表与详情预览（含 `image` 接口展示）。
4. 实现配置读写与摘要查询页面，完成基本可用性打磨。

## 验收标准

- 可按时间范围查询截图并查看 OCR 状态与文本。
- 可查看单条截图原图并与元数据联动展示。
- 可读取并更新动态配置，更新后可再次读取验证。
- 可按时间范围查询摘要列表并展示关键字段。

## 测试要求

- 前端组件测试覆盖关键交互（筛选、提交、错误提示）。
- 至少一条端到端 smoke 测试覆盖“查询截图 -> 查看图片”流程。
- 构建产物通过 `npm run build` 校验。
