## 目标

将 RapidOCR（GPU 加速）作为可选 OCR 引擎集成到 recall 新版，未安装时自动 fallback 到 Tesseract CLI。同时将 GPU 监控从 nvidia-smi CLI 升级为 pynvml。

## 范围

| # | 内容 | 文件 |
|---|------|------|
| 1 | OCR 引擎工厂函数 | `recall/services/ocr_engine.py`（新建） |
| 2 | Engine 集成工厂函数 | `recall/services/core/engine.py` |
| 3 | GPU 监控升级 pynvml | `recall/services/monitor/resource_monitor.py` |
| 4 | 可选依赖说明 | `recall/requirements.txt` |
| 5 | 测试 | `recall/tests/test_ocr_engine.py`（新建）、`recall/tests/test_resource_monitor.py` |

不包含：OCRWorker 本身的逻辑变更（已有 OCRCallable 注入机制，无需修改）。

## 实施计划（分步骤）

### 1. 新建 `recall/services/ocr_engine.py`

创建工厂函数 `create_ocr_engine() -> OCRCallable`：

- 尝试 `import rapidocr_onnxruntime` + `import onnxruntime`
  - 成功：检测 CUDAExecutionProvider，创建 RapidOCR 实例（参考 `recall_old/ocr_worker.py:34-55`）
  - 返回包装函数：PIL 加载图片 → numpy 数组 → RapidOCR 推理 → 拼接文本行
- ImportError 时 fallback：检测 tesseract 可用性，返回现有 `_default_ocr_engine`
- 两者都不可用时 raise RuntimeError

关键参考代码：
- `recall_old/ocr_worker.py:34-55` — RapidOCR 初始化（CUDA 检测 + 引擎创建）
- `recall_old/ocr_worker.py:57-89` — 单张处理（PIL→numpy→OCR→文本提取）
- `recall/services/ocr_worker.py:14` — `OCRCallable` 类型定义
- `recall/services/ocr_worker.py:17-34` — 现有 Tesseract 实现

### 2. 修改 `recall/services/core/engine.py`

Engine.__init__ 中，当未注入 ocr_worker 时：
```python
from recall.services.ocr_engine import create_ocr_engine
ocr_engine = create_ocr_engine()
self.ocr_worker = OCRWorker(ocr_engine=ocr_engine)
```

### 3. 升级 GPU 监控 `recall/services/monitor/resource_monitor.py`

修改 `_sample_gpu_usage()`：
- 尝试 `import pynvml`，用 `nvmlInit()` + `nvmlDeviceGetUtilizationRates()` 获取 GPU 利用率
- ImportError 时 fallback 到现有 nvidia-smi subprocess 方式
- 参考 `recall_old/utils/gpu.py` 的 pynvml 用法

### 4. 更新 `recall/requirements.txt`

添加可选依赖注释段：
```
# Optional: GPU-accelerated OCR
# pip install rapidocr-onnxruntime onnxruntime-gpu
# Optional: lightweight GPU monitoring
# pip install pynvml
```

### 5. 测试

**新建 `recall/tests/test_ocr_engine.py`**：
- mock rapidocr_onnxruntime import 成功 → 验证返回 RapidOCR 引擎
- mock rapidocr_onnxruntime import 失败 → 验证 fallback 到 tesseract
- 验证返回的函数符合 OCRCallable 签名

**修改 `recall/tests/test_resource_monitor.py`**：
- mock pynvml import 成功 → 验证使用 pynvml
- mock pynvml import 失败 → 验证 fallback 到 nvidia-smi

## 验收标准

- `create_ocr_engine()` 在有 RapidOCR 时返回 GPU 加速引擎，无时返回 Tesseract 引擎
- RapidOCR 引擎支持 CUDA 自动检测（有 GPU 用 GPU，无 GPU 用 CPU）
- GPU 监控优先使用 pynvml，不可用时 fallback nvidia-smi
- 现有测试全部通过，无回归
