# start - 开发工作流程

开始开发第一个待开发需求。

## 执行步骤

1. **读取需求**：从 `doc/REQUIREMENTS.md` 的"待开发"区域读取第一个需求，特别标注的不进行开发。
2. **进入 Plan 模式**：使用 EnterPlanMode 工具进入计划模式
3. **讨论细化**：在 plan 模式中与用户讨论，明确需求细节
4. **重写需求**：将细化后的需求写入 `doc/REQUIREMENTS.md` 的"开发中"区域
5. **开发实现**：退出 plan 模式，开始编码
6. **自测验证**：开发完成后自行测试，确保功能正常
7. **更新日志**：测试通过后更新 `doc/CHANGELOG.md`
8. **完成归档**：将需求从"开发中"移到"已完成"

## 注意事项

- 每次只进行一个功能的开发
- 遵循"先跑起来，再完善"的原则
- 最小实现，避免过度设计
