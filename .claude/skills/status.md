# /status — 任务看板

扫描 `.tasks/` 目录，展示当前所有任务的状态。

## 执行步骤

1. 扫描 `.tasks/backlog/`、`.tasks/wip/`（含子目录）、`.tasks/done/` 中的所有 `.md` 文件
2. 解析每个文件的 YAML front matter（`---` 之间的内容）
3. 判断状态：
   - 在 `backlog/` → 待领取（检查 depends_on 是否满足，不满足则标记为"阻塞"）
   - 在 `wip/` → 进行中（从子目录名获取 worker-id）
   - 在 `done/` → 已完成
4. 输出任务看板表格：

```
| ID  | 标题           | 优先级 | 状态     | Worker | 依赖   |
|-----|---------------|--------|---------|--------|--------|
```

5. 输出统计摘要：`待领取 X / 进行中 X / 已完成 X / 阻塞 X / 总计 X`
6. 如果有阻塞的任务，列出阻塞原因
