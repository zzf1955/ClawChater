## 背景

重构路线固定前端技术栈为 React + Vite + Tailwind，需要替代现有前端并承接新的 REST API。

## 目标

交付可用的前端界面，满足截图浏览、动态配置管理和 summary 查询三类核心使用场景。

## 范围

包含：
- 截图列表与详情查看（含图片加载）。
- 配置读取与编辑提交。
- summary 时间范围查询与展示。
- 基础状态管理、错误提示与加载态。

不包含：
- 复杂权限体系与多用户能力。

## 实施计划（分步骤）

1. 设计页面路由与模块结构（Browser/Config/Summary）。
2. 封装 API 客户端与错误处理中间层。
3. 实现截图浏览与详情组件。
4. 实现配置编辑表单与提交反馈。
5. 实现 summary 查询视图并联调后端接口。

## 验收标准

- [ ] 前端可完成三类核心操作且与新 API 对接成功。
- [ ] 异常与空数据场景有可理解的提示。
- [ ] 在桌面常见分辨率下布局可用。

## 测试要求

- [ ] 关键组件单元测试（至少覆盖 API 成功/失败两类路径）。
- [ ] 手工端到端验证截图浏览、配置保存、summary 查询。
