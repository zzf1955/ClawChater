## 背景

重构文档已经给出 API 基线，前端和外部 agent 都依赖统一 REST 接口访问 Recall 数据。

## 目标

基于 Fastify 实现核心 REST API，覆盖 OCR 查询、图片获取、summary 写入查询和动态配置读写。

## 范围

包含：
- `GET /api/ocr`
- `GET /api/screenshot/:id/image`
- `POST /api/summaries`
- `GET /api/summaries`
- `GET /api/config` / `POST /api/config`
- 统一错误响应、参数校验和分页/限流基线。

不包含：
- 前端页面与交互实现。

## 实施计划（分步骤）

1. 设计 API schema（query/body/response）并完成参数校验。
2. 对接数据库服务层，完成各 endpoint 业务逻辑。
3. 实现图片文件读取与安全路径校验。
4. 增加统一错误处理与访问日志。
5. 编写 API 集成测试与接口文档。

## 验收标准

- [ ] 文档定义的 API 全部可用并返回正确结构。
- [ ] 配置读写可实时生效（通过服务层验证）。
- [ ] 异常场景（参数错误、资源不存在）返回一致错误格式。

## 测试要求

- [ ] API 集成测试覆盖成功与失败路径。
- [ ] 手工验证关键接口联通（curl 或 Postman）。
