## 背景

当前仓库主体仍是 Python Recall 实现，重构决策已明确要求一次性转为 TypeScript，并固定后端 Fastify、前端 React + Vite + Tailwind、数据库 SQLite。

## 目标

建立可运行的 TS 基础工程，形成后续模块开发的统一脚手架与目录约定。

## 范围

包含：
- 创建 `recall-ts`（或等价）目录结构与 `package.json` 工程配置。
- 初始化 TypeScript 编译、lint、test 基础能力。
- 初始化 Fastify 服务入口与健康检查接口。
- 初始化 React + Vite + Tailwind 前端入口与页面骨架。

不包含：
- 具体截图/OCR/业务 API 细节实现。

## 实施计划（分步骤）

1. 设计新目录结构（`main/trigger/capture/resource-monitor/ocr-worker/api/frontend`）并落盘。
2. 初始化 Node/TypeScript 工具链（`tsconfig`、脚本命令、基础规范）。
3. 搭建 Fastify 最小可运行服务与 `/health` 基线接口。
4. 搭建 React + Vite + Tailwind 最小可运行前端，并配置代理到后端。
5. 补充工程 README（启动方式、目录说明、脚本说明）。
